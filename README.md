# Week 2 - Searching and Sorting Algorithms in Java

## Overview

This repository contains Java implementations of fundamental Searching and Sorting algorithms. The programs are designed using real-world problem scenarios to demonstrate how these algorithms can be applied in practical situations.

The repository includes detailed problem analysis, algorithm selection reasoning, execution flow, program structure analysis, and complexity analysis for each implementation.

## Searching Algorithms

### 1. Fraud Transaction Detection (Linear Search)

**Objective:**
Search for a suspicious transaction ID within transaction records.

**Concepts Covered:**
- Linear Search
- Sequential Traversal
- Boolean Flag Technique

**Key Features:**
- Works on unsorted data
- Checks elements one by one
- Simple and easy to implement

---

### 2. Server Log Search (Binary Search)

**Objective:**
Locate a specific server log ID in a sorted dataset.

**Concepts Covered:**
- Binary Search
- Divide and Conquer Strategy
- Search Space Reduction

**Key Features:**
- Requires sorted data
- Faster than Linear Search for large datasets
- Eliminates half of the remaining search space after each comparison

---

## Sorting Algorithms

### 3. Sort Employee Salaries (Bubble Sort)

**Objective:**
Arrange employee salary records in ascending order.

**Concepts Covered:**
- Bubble Sort
- Adjacent Element Comparison
- Swapping Technique

**Key Features:**
- Repeatedly compares neighboring elements
- Largest element moves to its correct position after each pass
- Easy to understand and implement

---

### 4. Sort Website Response Times (Insertion Sort)

**Objective:**
Sort website response times for performance analysis.

**Concepts Covered:**
- Insertion Sort
- Element Shifting
- Sorted and Unsorted Partition

**Key Features:**
- Builds the sorted array gradually
- Efficient for small datasets
- Performs well when data is partially sorted

---

### 5. Prioritize Bug Reports (Selection Sort)

**Objective:**
Arrange bug priorities in ascending order.

**Concepts Covered:**
- Selection Sort
- Minimum Element Selection
- Position Swapping

**Key Features:**
- Selects the smallest element during each pass
- Performs fewer swaps compared to Bubble Sort
- Simple sorting strategy

## Complexity Analysis

| Algorithm | Best Case | Average Case | Worst Case | Space Complexity |
|------------|------------|------------|------------|------------|
| Linear Search | O(1) | O(n) | O(n) | O(1) |
| Binary Search | O(1) | O(log n) | O(log n) | O(1) |
| Bubble Sort | O(n²) | O(n²) | O(n²) | O(1) |
| Insertion Sort | O(n) | O(n²) | O(n²) | O(1) |
| Selection Sort | O(n²) | O(n²) | O(n²) | O(1) |

## Key Takeaways

- Linear Search is suitable for unsorted datasets.
- Binary Search provides efficient searching on sorted data.
- Bubble Sort works through repeated adjacent comparisons.
- Insertion Sort builds a sorted sequence incrementally.
- Selection Sort repeatedly places the minimum element in its correct position.
- Understanding these algorithms forms the foundation for advanced Data Structures and Algorithms (DSA).
